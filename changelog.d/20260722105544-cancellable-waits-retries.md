Add cooperative cancellation across `wait`, `timeout`, `retry`, and `waitFor` via optional `AbortSignal` support.

- `wait(time, {signal})` — cancellable sleep. Rejects immediately if the signal is already aborted, clears its timer and removes its listener when the delay elapses or the signal aborts, and rejects with `signal.reason` (the platform's `AbortError` when aborted without an explicit reason). `wait(time)` is unchanged.
- `timeout({signal})` — composes an external signal with the deadline. Timeouts still reject with `TimeoutError`; external cancellation rejects with `signal.reason`. In both cases `control.signal` aborts with that same value, and every timer/listener is cleaned up on all paths. `control.check()` now honors an aborted `control.signal` first, so it throws the external cancellation reason immediately even when deadline time remains (and still throws `TimeoutError` when the deadline passes before its timer runs).
- `retry({signal, shouldRetry})` — `signal` prevents new attempts and cancels an in-progress retry delay (rejecting with `signal.reason`), and is composed into the per-attempt `TimeoutControl` when a timeout is set. Without a timeout the callback now receives a forward-compatible `{signal}` object so owned work can cooperate with cancellation, and a non-cooperative callback that resolves after an abort no longer returns stale success. `shouldRetry` is a forward-compatible object-argument predicate (`({error, tryNumber, tries, signal}) => boolean | Promise<boolean>`) that receives the external signal and may be async; a falsy result rethrows the current error without another attempt or delay, and a cancellation observed while it runs wins over that rethrow.
- `waitFor({signal})` — external cancellation is reflected in the callback's `control.signal`, retry delays are cancellation-aware, and no callback starts after cancellation. A callback that ignores its signal and resolves after cancellation or the deadline no longer wins — the cancellation reason or a `TimeoutError` is surfaced instead — and the run rejects with `signal.reason`.

All existing call forms and the `{control}` callback convention are preserved.
